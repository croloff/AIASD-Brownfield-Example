# Issue 010: Missing ClaimTypes.NameIdentifier in JWT Token

**Priority**: 🔴 **Critical**  
**Category**: Authentication / JWT  
**Status**: Open  
**Created**: 2026-05-21

## Summary

The JWT token generated by UserService.GetToken() does not include the `ClaimTypes.NameIdentifier` claim (user ID). This claim is essential for controllers to extract the authenticated user's ID and verify resource ownership.

## Impact

- **Severity**: Critical
- **Affected Components**: 
  - Services/Implementations/UserService.cs
  - Controllers/PostController.cs (cannot extract user ID from token)
  - Controllers/CommentController.cs (cannot extract user ID from token)
- **Scope**: Authorization checks for post/comment ownership will fail

## Current Behavior

```csharp
// UserService.GetToken() - MISSING NameIdentifier claim
var claims = new List<Claim>
{
    new Claim(ClaimTypes.Name, user.UserName),
    new Claim(ClaimTypes.Email, user.Email)
    
    // MISSING:
    // new Claim(ClaimTypes.NameIdentifier, user.Id)
};

var token = new JwtSecurityToken(
    issuer: jwtSettings.ValidIssuer,
    audience: jwtSettings.ValidAudience,
    claims: claims,
    expires: DateTime.Now.AddHours(3),
    signingCredentials: new SigningCredentials(key, SecurityAlgorithm.HmacSha256)
);
```

## Expected Behavior (Per Instructions)

Per **jwt-authentication.instructions.md** reference implementation:
```csharp
var claims = new List<Claim>
{
    new Claim(ClaimTypes.NameIdentifier, user.Id),  // User ID
    new Claim(ClaimTypes.Name, user.UserName),       // Username
    new Claim(ClaimTypes.Email, user.Email)          // Email
};
```

Per **PROJECT-REQUIREMENTS.md** § JWT Token Structure:
```
Payload:
{
  "sub": "{userId}",           // Subject (user ID) ← Required for authorization
  "iss": "PostHubAPI",
  "aud": "PostHubAPI-Client",
  "exp": {timestamp},
  "iat": {timestamp}
}
```

## Required Changes

### Services/Implementations/UserService.cs

Add the NameIdentifier claim in GetToken method:

```csharp
private string GetToken(User user, JwtSettings jwtSettings)
{
    var claims = new List<Claim>
    {
        new Claim(ClaimTypes.NameIdentifier, user.Id),    // ADD THIS
        new Claim(ClaimTypes.Name, user.UserName),
        new Claim(ClaimTypes.Email, user.Email)
    };
    
    // ... rest of token generation code
}
```

## How Controllers Will Use It

Once this claim is present, controllers can verify ownership:

```csharp
[HttpPut("{id}")]
[Authorize]
public async Task<IActionResult> EditPost(int id, EditPostDto editPostDto)
{
    var userId = User.FindFirst(ClaimTypes.NameIdentifier)?.Value;
    
    if (userId == null)
        return Unauthorized();
    
    var post = await _postService.GetPostByIdAsync(id);
    
    // Now possible because Post has UserId property:
    if (post.UserId != userId)
        return Forbid();
    
    // ... continue with edit
}
```
